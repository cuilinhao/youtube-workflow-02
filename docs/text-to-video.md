## 批量文生视频功能复制方案

- **目标**：在侧栏新增入口“批量文生视频”，并提供一个与“批量图生视频”完全相同的 UI 与功能页面，默认复用现有图转视频 API；未来有独立“文转视频”接口时再切换调用。
- **入口与路由**：
  - 侧栏位置：排在“批量图生视频”下方，保留“新品”徽标。
  - Tab/路由：新增 `text-to-video`，URL 形式 `/?tab=text-to-video`（与现有 tab 机制一致）。
- **文案与多语言**：
  - 中文：批量文生视频；英文：Text → Video。
  - 新增 i18n 键值，列表及主区域标题/副标题共用。
- **UI/功能复用**：
  - 直接使用现有 “批量图生视频” 的界面、表单字段、交互与列表，包含创建任务、批量操作、生成、编辑、状态轮询等。
  - 标题与路由 id 改为 `text-to-video`，其他逻辑保持一致。
- **API 与默认参数**：
  - 先复用当前图转视频接口（`api.addVideoTask` 等），请求参数、字段、默认配置（画幅、水印、回调、翻译/回退开关）保持一致。
  - 若后续提供专用“文转视频”接口，再替换调用入口即可。
- **返回路径**：
  - 新页内的返回/导航应指向 `/?tab=text-to-video`，避免跳回图生视频。

> 备注：如需差异化模型选项或文案，可在后续迭代新增可配置的标题/描述 props。当前版本以快速复制可用功能为优先。***
